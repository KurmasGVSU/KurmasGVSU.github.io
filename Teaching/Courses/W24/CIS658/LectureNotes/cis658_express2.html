  • Any questions?
  • Basic idea:
    • Route matches
    • Run code that generates content.
    • Several ways to generate content:
      • Return a string
      • Return the contents of a file
      • Run a template and return the results.

Convention vs. Configuration

  • Visit the ExpressJS web page (https://expressjs.com/) and notice that it claims to be “unopinionated”
  • Notice how everything in Express is “spelled out”:
    • Name of route and how they are mapped to code
    • Which template system to use
    • Where to look for files,
    • etc.
  • Similarly, in the use_mini_express example, each route has to be explicitly mapped to a function.
  • But what if we used a convention: A path foo should execute a function named foo_route
    • Look at make_routes
  • There are many potential conventions
    • Default template (pug or ejs)
    • Default location/directory for templates
