  • Now we want to grab data the “real” way: From an API server
  • show ColorListAPI
    • Very simple because it doesn’t have any authentication/login
    • Notice use of port 3001. (API and React need to use different ports)
    • Notice the use of res.json to send raw data
    • visit localhost:3001/colors and see raw .json returned.
  • fetch makes an network request and returns a promise with the response.
    • request is GET by default, but this can be configured.
    • Notice that handling .json is a two-step process. First then gets the response as text. Next, we need to parse it as JSON to create the desired JavaScript object.
    • Once we have the object, we can use the state hook to set the colors.
  • useEffect hook is code that is (potentially) run after every render (i.e., after every time components are redrawn — which happens every time state changes.)
    • useEffect calls fetchColors.
      • Notice the [] as the 2nd parameter. This is the list of objects the hook should watch, and only re-run when items in the list change.
      • We only want to fetch colors upon page load, so we pass an empty array.
      • See what happens if the brackets are removed.
