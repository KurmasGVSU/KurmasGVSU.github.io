End-to-End testing

  • Testing pyramid
    • Lots of Unit tests
    • Fewer integration tests
  • Cucumber is a platform for specifying end-to-end tests.
  • Used in BDD (Behavior Driven Development)
  • The Cucumber platform has several parts:
    1. Gherkin: the English-like language used to describe the desired behavior
      • When I visit the root page
      • Then I should see a list of authors
    2. A web driver: Code that manipulates a web browser
    3. Step Definitions that call driver methods based on Gherkin statements
      • Cucumber supports step definitions in many different languages
        • Ruby
        • JavaScript
        • Java
  • To use cucumber with Chrome, you will need to download the driver
    • https://chromedriver.chromium.org/downloads
    • On macOS you may have to override the quarantine:
      • xattr -d com.apple.quarantine chromedriver

To set up Cucumber using Ruby:

  1. Create a Gemfile
  2. Add cucumber: gem 'cucumber'
  3. run bundle install (or bundle install --path vendor/bundle if you donâ€™t want to install Cucumber globally )
  4. Run bundle exec cucumber --init to initialize the features directory.

  5. Create a file (e.g., test.feature) and add a Scenario

    Feature: First test

  Scenario: Visit the root page
    When I visit the home page
    Then I should see the list of author
  6. Run bundle exec cucumber
    • You should see the result of the tests.

Cucumber and Rails

  • Instructions from https://cucumber.io/docs/installation/ruby/
  1. Add cucumber-rails and database_cleaner to your the :test section of your Gemfile
    • gem 'cucumber-rails', require: false
    • gem 'database_cleaner'
  2. Install the gems: bundle
  3. Run the generator: rails generate cucumber:install
