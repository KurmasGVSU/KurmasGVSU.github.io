Condition Variables

  • How can we efficiently wait for a thread to finish?
    • Spin wait is obviously a bad idea.
  • Idea: A queue of blocked processes similar to the locks we just discussed.
  • A condition variable has two operations: wait and signal
  • Figure 30.3
    • Notice that a lock must be held when calling signal or wait
    • Notice that the condition is just “regular” code.
    • Notice that the condition is checked in a while loop, not an if.
  • {. q} Why is a lock necessary? What could go wrong in Figure 30.5
    • Parent runs if (done == 0), then swapped out.
    • Child runs done = 1; signal (but no thread is listening)
    • Parent resumes and calls wait but no thread will now signal.
  • Technically a lock is not always required, but it is best to hold one and avoid subtle bugs.
    • The posix condition variable effectively requires the lock.

Bounded Buffer / Producer Consumer

  • One process adds to a buffer, the other removes from it.
  • Pipes do this.
  • Your pzip project could do this.
  • Figure 30.6: A buffer that can hold only one element (count is either 1 or 0.)
  • Why doesn’t figure 30.7 work?
    • Nothing preventing adding or removing more than one at a time.
  • Possible solution: Figure 30.8.
  • Does 30.8 work?
    • Yes, if there is a single producer and consumer.
    • No if there are multiple consumers.
      1. Consumer 1 waits because buffer is empty.
      2. Producer fills buffer and signals.
      3. Consumer 2 happens to be running and calls get.
      4. Consumer 1 wakes up and moves on, but there is nothing to get.
  • Bottom line: A signal suggests the state may have changed, it does not guarantee it.
    • This is called Mesa semantics.
  • If you guarantee the woken thread can run, it is called Hoare sematics.
    • This is much more difficult. Almost every system uses Mesa semantics.
  • Changing if to while fixes this first problem.
  • What is the other problem?
    • One consumer can wake another (See Figure 30.11).
  • Solution: Separate empty and full conditions.
  • Now we can add more buffer slots.
