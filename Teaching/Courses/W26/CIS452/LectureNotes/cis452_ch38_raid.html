RAID

  • Scaling up single disks is difficult and risky
    • Hard to keep larger disks spinning fast.
    • A failure can destroy a lot of data.
  • Solution: An array of disks working together.
  • Transparent: A RAID cluster can present the same interface as a single disk. No need to update OS to use RAID.
  • Simply striping data across disks is dangerous
    • If any one disk fails, all data lost.

Raid Levels:

  • Raid 0: Striping.
    • Good performance
    • No redundancy
    • There is a tradeoff between chunk size and speed.
      • small chunks maximize parallel access
      • Large chunks allow multiple requests to run in parallel
        (which means seek times can run in parallel)
  • Raid 1: Mirroring
    • How many failures can be tolerated?
      • 1 guaranteed, but could be more if they are on the same “half”
    • How does the writing speed of Raid 1 compare to a single disk?
      • More or less the same. (Slower of two disks)
    • How about the reading speed for a sequential workload?
      • Actually about the same. Even though you can read two blocks at once, it takes about as much time to seek over a block as just to read it.
    • How about the reading speed for a random workload?
      • There is a speedup here because the seeks can happen in parallel.

  • Raid 4: Parity
    • Reading speedup of N - 1
    • Can tolerate 1 disk failure (never more)
    • What must happen to write a single block?
      • We need to also update the parity block.
    • Must we read all the blocks in the stripe?
      • No. Just load target block and parity block. (“subtractive”)
    • What is the biggest performance problem?
      • Parity disk is a “hot spot”

  • Raid 5
    • Rotate the parity blocks around the stripe.
    • Raid 5 has what is called the “small write penalty”
    • Why are random small writes much worse than large sequential writes?
      • Each small random write requires two reads and two writes. Sequential writes don’t have “extra” reads: If all blocks in a stripe are written, then we can calculate the parity from the data to be written.
