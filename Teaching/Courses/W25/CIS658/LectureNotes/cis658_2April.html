Promises

  • Start with ajax1.html.
  • Suppose we need to make multiple Ajax calls, and that each one depends on the previous
    • e.g., Need to get patientID from userDB, then need to send patientID to EPIC service, then need to send resultID to lab service.
    • Add status check and else with error checking.
    • Add second call nested in the first (with error checking.)
    • Add a third call nested in the second.
  • Obviously this causes “nested callback hell” — especially given error checking.
    • Error handling can be especially redundant
    • Factoring out callback into named functions can help somewhat.
  • A better approach is to use a Promise.
    • Promises wrap up callbacks in an object so you can chain them rather than nest them.
