Data Structures

  • Think of an ArrayList as simply a tool for storing stuff (i.e., Java Objects).
  • Forget about how these things work and focus on what they do.
  • What are the fundamental operations on an ArrayList?
    • get(x)
    • put(x)
  • What are some other common things we do with ArrayLists?
    • add (to the end)
    • removeFromEnd
    • insert(object, location) (insert at a specific location)
    • remove(location)
  • What is the running time of each?
    • get: O(1)
    • put: O(1)
    • add: O(1)
    • removeFromEnd: O(1)
    • insert(object, location): O(n)
    • remove(location): O(n)
  • This is good if most of of your operations are O(1); not so good if you are doing a lot of inserts and removes.
  • Consider a priority queuing system for course overrides:
    • Seniors go to the head of the list because they need to graduate.
    • Juniors get in line behind the seniors.
  • If n students sign up for overrides, it could make this process O(n^2).
  • Think about this, however: The fact that get and put is O(1) is not helpful because we don’t use these operations. Is there a way to make the operations we do use faster at the expense of the ones we aren’t using.
  • ArrayLists focus on an item’s specific position (what’s, first, what’s second, etc.)

  • Collection: “an object that gathers and organizes other objects”
    • List, Map, Set, Tree, etc.
  • Data Type: “a group of values and the operations defined on those values”.
    • Specific collections (e.g., Lists)
  • Abstract Data Type: Data type where the implementation details are not specified
    • A List is an Abstract Data Type since implementation details are not given
    • ArrayLists and LinkedLists are not abstract because the implementation details are relevant here.
