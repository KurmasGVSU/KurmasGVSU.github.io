Deadlock

  • Cycle of locks allows deadlock.
  • Not always obvious to see (e.g., at the OS-level)
    • VM may need to access the file system
    • File system may request VM page.
    • encapsulation can hide who is asking for which locks.
  • Consider Vector v1, v2; v1.AddAll(v2)
    • What if this locks the vectors, then another thread did v2.AddAll(v1)?
  • Four necessary conditions:
    1. Mutual exclusion
    2. Hold-and-wait
    3. No preemption
    4. Circular wait

  • If any of above are not met, deadlock is not possible.

  • Circular Wait
    • How to avoid?
    • Total order of locks.
      • Sometimes a partial order is sufficient.
      • Remember: order is a convention. Programmers can still screw it up.
  • Hold-and-wait
    • How to avoid?
      • Put a lock around groups of lock acquisition.
      • Why does this work?
      • What are the costs of this solution?
  • No-Preemption
    • How to avoid?
      • Release if you can’t get them all
    • Livelock is a possible, but unlikely scenario.
    • This doesn’t technically add preemption. The locks are given up voluntarily.
